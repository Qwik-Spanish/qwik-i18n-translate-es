# Vite

Qwik aprovecha [Vite](https://vitejs.dev/) para brindar una experiencia de desarrollo rápida. Vite es una herramienta de construcción que sirve tu código a través de módulos ES nativos durante el desarrollo.
Esto significa que no necesita empaquetar ni transpilar tu código para que funcione en el navegador. También cuenta con una sustitución de módulos en caliente (HMR) extremadamente rápida que actualiza tu navegador al instante cuando realizas cambios en tu código.

## Configuración

Después de crear un proyecto de Qwik, encontrarás un archivo `vite.config.js` en la raíz de tu proyecto. Aquí es donde puedes configurar Vite.

## Plugins

Qwik incluye dos plugins que facilitan el uso de Vite con Qwik y Qwik City.

```ts title="vite.config.ts"
import { defineConfig } from 'vite';
import { qwikCity } from '@builder.io/qwik-city/vite';
import { qwikVite } from '@builder.io/qwik/optimizer';
import tsconfigPaths from 'vite-tsconfig-paths';

export default defineConfig(() => {
  return {
    plugins: [qwikCity(), qwikVite(), tsconfigPaths()],
  };
});
```

### `qwikVite()`

Para modificar la configuración, puedes pasar un objeto a la función `qwikVite`. Las opciones posibles son:

#### `debug`

```js
/**
 * Imprime registros de depuración detallados del plugin Qwik.
 * Valor predeterminado: `false`
 */
debug?: boolean;
```

#### `entryStrategy`

```js
/**
 * La entry strategy de Qwik que se utilizará al empaquetar para producción.
 * Durante el desarrollo, el tipo siempre es `hook`.
 * Valor predeterminado: `{ type: "smart" }`
 */
entryStrategy?: EntryStrategy;
```

#### `srcDir`

```js
/**
 * El directorio para encontrar todos los componentes de Qwik. Dado que Qwik
 * no tiene una entrada única, el `srcDir` se utiliza para buscar archivos de Qwik de forma recursiva.
 * Valor predeterminado: `src`
 */
srcDir?: string;
```

#### `vendorRoots`

```js
/**
 * Lista de directorios para buscar componentes de Qwik o proveedores de forma recursiva.
 * Valor predeterminado: `[]`
 */
vendorRoots?: string[];
```

#### `client`

```js
client?: {
    /**
     * El punto de entrada para las compilaciones del cliente. Por lo general, esto sería
     * el componente principal de la aplicación.
     * Valor predeterminado: `src/components/app/app.tsx`
     */
    input?: string[] | string;
    /**
     * Punto de entrada para el desarrollo solo en el lado del cliente con recarga de módulos en caliente (HMR).
     * Esto es solo para el desarrollo con Vite y no utiliza SSR.
     * Valor predeterminado: `src/entry.dev.tsx`
     */
    devInput?: string;
    /**
     * Directorio de salida para la compilación del cliente.
     * Valor predeterminado: `dist`
     */
    outDir?:

 string;
    /**
     * La compilación del cliente creará un manifiesto y este hook
     * se llamará con los datos de compilación generados.
     * Valor predeterminado: `undefined`
     */
    manifestOutput?: (manifest: QwikManifest) => Promise<void> | void;
};
```

#### `ssr`

```js
ssr?: {
    /**
     * El punto de entrada para el renderizador SSR. Este archivo debería exportar
     * una función `render()`. Este punto de entrada y la función de exportación `render()`
     * también se utilizan para el desarrollo SSR de Vite y el modo de depuración de Node.js.
     * Valor predeterminado: `src/entry.ssr.tsx`
     */
    input?: string;
    /**
     * Directorio de salida para la compilación del servidor.
     * Valor predeterminado: `dist`
     */
    outDir?: string;
    /**
     * La compilación SSR requiere el manifiesto generado durante la compilación del cliente.
     * De forma predeterminada, este complemento vinculará el manifiesto del cliente a la compilación SSR.
     * Sin embargo, la opción `manifestInput` se puede utilizar para proporcionar manualmente un manifiesto.
     * Valor predeterminado: `undefined`
     */
    manifestInput?: QwikManifest;
};
```

#### `optimizerOptions`

```js
/**
 * Opciones para el optimizador de Qwik.
 * Valor predeterminado: `undefined`
 */
optimizerOptions?: OptimizerOptions;
```

#### `transformedModuleOutput`

```js
/**
 * Hook que se llama después de la compilación y proporciona todos los módulos transformados
 * que se utilizaron antes de la empaquetación.
 */
transformedModuleOutput?:
    | ((transformedModules: TransformModule[]) => Promise<void> | void)
    | null;
```

#### `devTools`

```js
devTools?: {
    /**
     * Mantén presionadas las teclas definidas para habilitar el inspector de desarrollo de Qwik.
     * De forma predeterminada, el comportamiento se activa al presionar la tecla `Alt` izquierda o derecha.
     * Si se establece en `false`, el inspector de desarrollo de Qwik estará deshabilitado.
     * Los valores válidos son los valores de `KeyboardEvent.code`.
     * Ten en cuenta que los sufijos 'Left' y 'Right' se ignoran.
     */
    clickToSource: string[] | false;
};
```

### `qwikCity()`

Para modificar la configuración, puedes pasar un objeto a la función `qwikCity`. Las opciones posibles son:

#### `routesDir`

```js
/**
 * Directorio de las `rutas`. Valor predeterminado: `src/routes`.
 */
routesDir?: string;
```

#### `serverPluginsDir`

```js
/**
 * Directorio de los `plugins del servidor`. Valor predeterminado: `src/server-plugins`.
 */
serverPluginsDir?: string;
```

#### `basePathname`

```js
/**
 * El nombre base de la ruta se utiliza para crear rutas de URL absolutas hasta
 * el `hostname`, y siempre debe comenzar y terminar con un `/`.
 * Valor predeterminado: `/`.
 */
basePathname?: string;
```

#### `trailingSlash`

```js
/**
 * Asegura que haya una barra diagonal al final de las URL de página. Valor predeterminado: `true`.
 * (Nota: Las versiones anteriores

 tenían el valor predeterminado `false`).
 */
trailingSlash?: boolean;
```

#### `mdxPlugins`

```js
/**
 * Habilita o deshabilita los plugins de MDX incluidos de forma predeterminada en qwik-city.
 */
mdxPlugins?: MdxPlugins;
```

#### `mdx`

```js
/**
* Opciones de MDX https://mdxjs.com/
*/
mdx?: any;
```

#### `platform`

```js
/**
* El objeto de plataforma que se puede utilizar para simular las vinculaciones de Cloudflare.
*/
platform?: Record<string, unknown>;
```

## Solución de problemas

Antes de crear un problema en el repositorio de Qwik, verifica la sección "Solución de problemas" en la [documentación de Vite](https://vitejs.dev/guide/troubleshooting.html) y asegúrate de estar utilizando la versión recomendada.
