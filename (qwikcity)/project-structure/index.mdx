# Estructura del proyecto

Un proyecto típico de Qwik se ve así:

```bash
qwik-app-demo
├── README.md
├── package.json
├── public
│   └── favicon.svg
├── src
│   ├── components
│   │   └── router-head
│   │       └── router-head.tsx
│   ├── entry.ssr.tsx
│   ├── global.css
│   ├── root.tsx
│   └── routes
│       ├── flower
│       │   ├── flower.css
│       │   └── index.tsx
│       ├── index.tsx
│       ├── layout.tsx
│       └── service-worker.ts
├── tsconfig.json
└── vite.config.ts
```

## Archivos del proyecto

### `src/routes/`

El directorio `src/routes/` es un directorio especial en Qwik City, ya que es el directorio donde Qwik City buscará tus páginas. Las carpetas y archivos dentro de este directorio tienen un significado especial y se mapearán a la URL de tu aplicación.

- `src/routes/index.tsx` es la página de inicio de tu aplicación.
- `src/routes/layout.tsx` es el esquema raíz (root layout) de tu aplicación, todas las páginas se renderizarán dentro de este layout.

> Consulta la sección de [Enrutamiento](/docs/routing) para obtener más información.

### `src/components/`

El directorio `src/components/` es un directorio por convención. Todos los proyectos de Qwik tendrán este directorio, pero puedes cambiarlo si lo deseas. Aquí es donde debes poner tus componentes, es decir, piezas de código reutilizables que se pueden usar en varios lugares. No son rutas ni layouts, pero se pueden usar dentro de ellos.

Por ejemplo, un componente `Button` debería estar dentro de `src/components/button/button.tsx`.

### `public/`

El directorio `public/` contiene activos estáticos (static assets) como imágenes, fuentes e iconos. Cuando compilas tu aplicación, estos archivos se copiarán al directorio `dist/` y se servirán en la raíz.

Consulta la [configuración de Vite para más información](https://vitejs.dev/guide/build.html#public-base-path).

### `src/entry.ssr.tsx`

Punto de entrada SSR (rendizado del lado del servidor). En todos los casos en que la aplicación se renderiza fuera del navegador, este punto de entrada (entry point) será el común.

- Servidor (express, cloudflare...)
- npm run start
- npm run preview
- npm run build

### `src/root.tsx`

El archivo `src/root.tsx` es el punto de entrada del árbol de la aplicación. Es el primer componente que se renderizará. Es la raíz del árbol.

### `src/global.css`

El archivo `src/global.css` es un archivo CSS global. Si necesitas definir algún CSS que se utiliza en varios lugares de tu aplicación, puedes definirlo aquí.

El componente raíz (`src/root.tsx`) importa este archivo de forma predeterminada.

### `tsconfig.json`

El archivo `tsconfig.json` contiene la configuración del compilador de TypeScript. Es estándar para cualquier proyecto de TypeScript.

### `vite.config.ts`

Qwik utiliza Vite para construir el proyecto. El archivo `vite.config.ts` contiene la configuración de Vite. Es estándar para cualquier proyecto de Vite. Consulta la [documentación de Vite](https://vitejs.dev/config/) para obtener más información.
